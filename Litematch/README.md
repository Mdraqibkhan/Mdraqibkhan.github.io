# LiteMatch project page

A static academic project page styled after the clean, centered presentation commonly used by computer-vision papers.

## Files

- `index.html` — page content
- `style.css` — responsive styling
- `script.js` — BibTeX copy button
- `assets/` — project figures and fallback placeholders

## Before publishing

1. Replace author names, homepages, affiliations, and BibTeX author list.
2. Verify the venue/acceptance wording before making the page public.
3. Add the final figures using these exact names:
   - `assets/front_visual.jpg`
   - `assets/final_model.jpg`
   - `assets/visual_all.jpg`
4. Confirm the paper and code URLs.

## Publish with GitHub Pages

1. Create a repository, for example `Litematch-project-page`.
2. Upload all files from this folder to the repository root.
3. Open **Settings → Pages**.
4. Under **Build and deployment**, select **Deploy from a branch**.
5. Choose the `main` branch and `/ (root)`, then save.

For a user/organization homepage, name the repository `YOUR_USERNAME.github.io`.
